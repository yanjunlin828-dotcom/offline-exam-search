## 1. 最终要求与验收红线

**一句话**：离线、纯 Python 标准库、保证能启动地，对一个文件夹里的 `.md`/`.ipynb` 做高精度检索，能区分"概念定义"与"代码实现"，支持中文两字概念。

**硬约束（验收红线，违反即不合格）：**

| 约束 | 落地为 |
|---|---|
| 无网络 | 前端 JS（marked/highlight）考前下载、打包进 `web/`，HTML 里**只引用相对路径，禁止任何 CDN** |
| 考试机不可控 | 引擎**只用标准库**，零 pip、零编译期特性（不碰 FTS5/trigram） |
| 中文短概念 | 分词用**字符 bigram**（覆盖 2 字词），不用 trigram |
| `.ipynb` 是 JSON | 按 cell 解析，不能当纯文本 grep |
| 防注入 | 查询解析全在 Python 内做，用户输入只当普通字符切词 |

**功能验收（§7 金标准用例必须全过）：**
- `ADMM 定义` → 定义 prose 块 Top-1，prose 档不出现实现代码
- `范数` / `凸性`（两字中文）→ 能召回
- `交替方向乘子法` / 全称 → 与 `ADMM` 同组，但只含 `method` 的无关块**不**出现
- `ADMM 实现` → 优先 code 块
- `ADMM AND "x*` → 不报错
- 代码块里的 `# 定义 ...` → **不**被当标题
- 大小写 / 中英混写 → 不影响召回
- 单次查询 < 1 秒；**且在接近考试环境的机器上完整跑通构建+查询（不可跳过）**

## 2. 已定型的全部决策（实现期不得擅自更改）

| # | 决策点 | **最终决定** | 理由 |
|---|---|---|---|
| A | chunk 切块粒度 | 一个章节内：**每个围栏代码块 = 1 个 code chunk**；代码块之间的连续散文**按空行分段，每段 = 1 个 prose chunk**。`position` 在所属章节内从 0 递增（0 = 紧跟标题的第一块）。空块丢弃。 | 让 `W_pos`（越靠标题越像定义）有意义；块够小，BM25 与片段展示都准 |
| B | 别名对打分的贡献 | 别名**只用于扩召回**。BM25 计分时，只对"**完整命中**某条别名短语"的块，把该短语的 token 计入评分集合；**部分命中的别名 token 一律不计分**。 | 防止"只含 method"的无关块被引噪（金标准专设陷阱） |
| C | 高亮在前端还是后端 | **前端做高亮**。后端只回原始 `text` / `heading_path` / `file` / `ctype` / `score`。 | 逻辑集中、好调；后端保持纯数据 |
| D | chunk 存储字段 | 用 **`tf`（`{token: 次数}` dict）+ `dl`（token 总数 int）** 替代方案 v2 §6 里的 `body_tokens`。`heading_tokens` 存为 set。 | 查询期算 BM25 O(\|Q\|) 即可，不必每次重数；这是对 §6 的有意微调 |
| E | 意图识别的输入 | `detect_intent` 接**原始查询字符串**，按子串匹配意图词，命中后**从查询串里剔除意图词**再切主题 token。 | `是什么`/`怎么写` 这类词被 bigram 切碎后无法整体匹配；且意图词若进检索会污染候选 |
| F | Python 版本基线 | 目标 **Python 3.7+**。**禁止**用海象运算符 `:=`（3.8）、`match`（3.10）、`X \| Y` 类型注解（3.10）。chunk 一律用 **plain dict**（不用 dataclass），与 pickle 结构一致。 | 考试机版本不可控，取最保守可用集 |
| G | CJK 范围 / unigram | 基线 `cjk_ext=False`（仅基本区 `U+4E00–U+9FFF`）、`enable_unigram=False`。两开关写进 `index["config"]`，查询期从中读取，保证索引/查询切词一致。 | 简单优先；开关留给 P2 |
| H | marked / highlight 的用法 | **P1 结果片段不走 marked 整段渲染**：prose 片段做"先转义→字面子串包 `<mark>`"；code 片段交 highlight.js 上色、**不再叠 `<mark>`**。marked.js 仍本地打包，留作 P2 的"查看完整块"预览。 | 避免 markdown 渲染与字面高亮互相破坏；同时满足"JS 全本地"红线 |
| I | 片段与分组 | 后端返回整块 `text`（块本就小）。前端：结果**按文件分组、可折叠**，每条顶部显示面包屑 `文件 > 章 > 节`。长块由前端做"首个命中处开窗"。 | 块小、简单、够用 |
| J | aliases.txt 格式 | 每行一组，写法间用 `=` 分隔；`#` 开头为注释行；空行忽略。见 §9。 | 人工维护最省事 |

## 5. 模块契约（数据结构 + 函数签名，必须严格遵守）

### 5.1 索引产物结构（`index.pkl` 内是一个 dict）

```python
index = {
    "version": 1,
    "config": {"enable_unigram": False, "cjk_ext": False},  # 查询期 tokenize 的开关，必须随索引存
    "chunks": [chunk, ...],          # chunk["id"] == 在列表中的下标
    "inverted": {token: [(chunk_id, tf), ...]},
    "df": {token: int},              # len(inverted[token])
    "N": int,                        # == len(chunks)
    "avgdl": float,                  # 所有块 dl 的平均
}

chunk = {
    "id": int,
    "file": str,                     # 相对路径，统一用 "/" 分隔
    "heading": str,                  # 本块所属标题文本
    "heading_path": str,             # 如 "优化方法 > ADMM > 定义"
    "text": str,                     # 原始正文（片段展示 + 前端高亮用）
    "heading_tokens": set,           # heading 经 tokenize 得到的 token 集（标题加分用）
    "ctype": str,                    # "prose" | "code"
    "position": int,                 # 本块在所属章节内的序号（0 = 紧跟标题）
    "tf": dict,                      # {token: 次数}，由 text+heading 的 token 统计（决策 D）
    "dl": int,                       # = sum(tf.values())，该块 token 总数
}
```

> 注意：`tf` 由「正文 token + 标题 token」共同统计（方案 v2 §7.1 要求标题命中也贡献 BM25）。`heading_tokens` 单独再存一份 set 仅用于 §7.2 的标题加分。

#### Task 0.1 `tokenizer.py` + 单测

**目标**：实现唯一真源分词器。

**要实现（规则）**：
1. 先 `text.lower()`（只影响英文/数字，中文无副作用）。
2. 顺序扫描字符，按"类别"分段：
   - 连续 `[a-z0-9]+` → 输出一个**英文词 token**。
   - 连续 CJK（`0x4E00 <= ord(c) <= 0x9FFF`；`cjk_ext=True` 时再并入 `0x3400–0x4DBF`）→ 输出**重叠 bigram**（`范数学` → `范数`,`数学`，即 i 从 0 到 L-2）；该段长度为 1 时输出该单字。`enable_unigram=True` 时**额外**为每个汉字补一个 unigram token。
   - 其余字符（标点、空白、全角符号、下划线、连字符等）→ 分隔符，丢弃。
3. 类别切换处必须断开（`ADMM算法` → `["admm","算法"]`）。

**验收（`tests/test_tokenizer.py`）**：
- `tokenize("范数学")` == `["范数","数学"]`
- `tokenize("ADMM算法")` == `["admm","算法"]`
- `tokenize("凸")` == `["凸"]`（单字段输出单字）
- `tokenize('ADMM AND "x*')` 不抛异常，`and`/`x` 作为普通词出现
- `tokenize("L1-norm")` 中 `-` 作为分隔符（确认 spec 取舍）

**易错点**：
- **bigram 是重叠窗口**，不是不重叠切分。
- **CJK 与 ASCII 边界必须切开**，用 `ord(c)` 判类别，类别变就结束当前段。
- 这是唯一真源，**别处不准再写第二份切词**——高亮里若需要切词也只调它。

**给 Claude Code 的 prompt**：
```
实现 tokenizer.py 中的 tokenize(text, enable_unigram=False, cjk_ext=False)，规则：
1) 先 text.lower()；
2) 连续 [a-z0-9]+ 作为一个英文词 token；
3) 连续 CJK（默认仅 U+4E00–U+9FFF；cjk_ext=True 再含 U+3400–U+4DBF）输出"重叠 bigram"
   （如 范数学 -> 范数,数学，即相邻两字一组，窗口重叠），该连续段长度为 1 时输出单字；
   enable_unigram=True 时额外为每个汉字补一个单字 token；
4) 其余字符（含标点/空白/下划线/连字符/全角符号）为分隔符丢弃；
5) 类别切换处断开，ADMM算法 -> ["admm","算法"]。
用 ord(c) 判类别，纯标准库 re/字符遍历皆可，Python 3.7+，不要用海象运算符。
再写 tests/test_tokenizer.py 覆盖：范数学、ADMM算法、单字"凸"、'ADMM AND "x*' 不报错、L1-norm。
```

#### Task 0.2 `parser.py` + 单测

**目标**：把 `.md`/`.ipynb` 切成 chunk 雏形（不分词）。

**要实现**：
- **Markdown 围栏状态机**：逐行扫，维护 `in_fence`。
  - 进入：`in_fence=False` 且某行去前导空格后以 ` ``` ` 或 `~~~` 开头（其后可跟语言标注）。记住围栏字符与长度。
  - 退出：`in_fence=True` 且某行是**同种字符、长度 ≥ 开围栏、整行仅围栏字符**的纯围栏行。
  - `in_fence=True` 期间**完全关闭**标题识别。
  - 仅 `in_fence=False` 时，`^#{1,6}\s` 才算 ATX 标题；维护标题栈（遇 level=L 标题，弹出栈内所有 level≥L 再压栈），生成 `heading_path`（` > ` 连接）。
  - 切块按**决策 A**：每个围栏块 = 1 个 `code` chunk；代码块之间按空行分段的散文 = `prose` chunk；`position` 在章节内从 0 递增。空块丢弃。
- **ipynb（JSON）**：
  - `json.loads` 读，遍历 `cells`；`source` 兼容 str / list，统一 `"".join(...)`。
  - `markdown` cell：**跑与 .md 完全相同的标题解析**，且**标题状态跨 cell 延续**（整个 notebook 当一篇连续文档）。
  - `code` cell：标 `code`，**只取 source、忽略 outputs**。
  - `raw` cell：跳过。
  - 老格式 guard：顶层无 `cells` 但有 `worksheets` 时，从 `worksheets[].cells` 取。
  - 单文件 JSON 解析失败：`except json.JSONDecodeError as e` 打印文件名后跳过该文件，**不得 `except: pass`，不得中断整次构建**。

**验收（`tests/test_parser.py`）**：
- 代码块内含 `# 定义 ADMM` 的 md → 该行**不**产生标题，整段为 1 个 code chunk。
- `~~~` 围栏不能被 ` ``` ` 闭合；代码里出现两个反引号不误判闭合。
- 多级标题的 `heading_path` 不串层（如 `一 > 1.1 > 名词` 正确）。
- ipynb：`source` 为 list 与 str 都能解析；markdown cell 的标题能延续到后续 cell。

**易错点**：
- **先判围栏、再判标题**（方案 v2 §15.2）。
- 闭合围栏的三条件缺一不可。
- ipynb 标题状态跨 cell 延续（容易写成每 cell 清栈）。
- 文件统一 `encoding="utf-8"`；坏文件按文件兜底跳过并告警。

**给 Claude Code 的 prompt**：
```
实现 parser.py：parse_markdown(text, file) 与 parse_notebook(text, file)，均返回
list[dict]，每个 dict 含 file/heading/heading_path/text/ctype/position（不含 tf/dl/id）。
要求：
- Markdown 用"围栏状态机"：维护 in_fence；开围栏=去前导空格后以 ``` 或 ~~~ 开头（记字符与长度）；
  闭围栏=同种字符、长度≥开围栏、整行仅围栏字符；in_fence 期间完全关闭标题识别；
  仅 in_fence=False 时 ^#{1,6}\s 才算标题，维护标题栈（遇 level L 先弹出栈内所有 ≥L 再压栈）生成 heading_path。
- 切块：每个围栏块=1个 code chunk；代码块之间按空行分段的散文=prose chunk；position 在章节内从 0 递增；空块丢弃。
- ipynb：json 解析，source 兼容 str/list 用 "".join；markdown cell 跑同一套标题解析且标题状态跨 cell 延续；
  code cell 标 code 只取 source 忽略 outputs；raw cell 跳过；顶层无 cells 但有 worksheets 时取 worksheets[].cells；
  单文件 JSON 解析失败 except json.JSONDecodeError 打印文件名后跳过，禁止 except: pass。
Python 3.7+，纯标准库。再写 tests/test_parser.py，务必含"代码块里有 # 定义 ADMM 不被当标题"的用例。
```

#### Task 0.3 `build_index.py`

**目标**：解析 → 分词 → 建倒排 → 落盘。

**要实现**：
- 递归遍历 `materials/`，筛 `.md`/`.ipynb`；**对文件路径排序后**再编号（id 可复现）；存相对路径，`path.replace("\\", "/")`。
- 对每个 chunk 雏形：`tokenize(text)` 得正文 token，`tokenize(heading)` 得标题 token；`tf = Counter(正文token + 标题token)`，`dl = sum(tf.values())`，`heading_tokens = set(标题token)`。空块（`dl==0`）丢弃。
- 建 `inverted`（`token -> [(id, tf)]`）、`df`（`len(postings)`）、`N`、`avgdl`。
- 写 `index` dict（含 `version`/`config`）到 `index.pkl`（`"wb"`）。
- 末尾打印：文件数、块数、token 种类数、avgdl，做 sanity check。

**验收**：在真实 `materials/` 上跑通；打印的块数/avgdl 合理；`index.pkl` 能被 `pickle.load` 回来且结构符合 §5.1。

**易错点**：
- **空块丢弃**，否则 `dl=0` 干扰 avgdl 与打分。
- `tf` 含标题 token（方案 v2 要求标题命中也贡献 BM25）。
- 路径统一正斜杠；统一 utf-8。
- `config` 必须写入（查询期要读它来切词，决策 G）。

**给 Claude Code 的 prompt**：
```
实现 build_index.py 的 build(materials_dir, out_path="index.pkl")：
- os.walk 递归找 .md/.ipynb，对相对路径排序后编号 id，路径用 / 分隔；
- 调 parser.parse_markdown/parse_notebook 得 chunk 雏形；
- 对每块：tokenize(text) + tokenize(heading) 合并用 collections.Counter 得 tf，dl=sum(tf.values())，
  heading_tokens=set(tokenize(heading))；dl==0 的空块丢弃；
- 组装 chunk dict（见契约），建 inverted={token:[(id,tf)]}、df={token:len(postings)}、N=len(chunks)、avgdl=平均 dl；
- 写 index={version:1, config:{enable_unigram:False,cjk_ext:False}, chunks, inverted, df, N, avgdl} 到 index.pkl(wb)；
- 末尾打印 文件数/块数/token种类/avgdl。
统一 encoding="utf-8"，纯标准库，Python 3.7+。

#### Task 0.4 `ranking.py`（P0 部分：BM25 + 标题/位置加分 + 手动类型过滤）

**目标**：实现核心相关度，CLI 可直接用。意图与别名留到 P1。

**要实现（P0）**：
- `bm25(index, chunk, qtokens)`：Lucene 式 IDF
  `IDF(t)=ln(1+(N-df(t)+0.5)/(df(t)+0.5))`，
  `score += IDF(t)*tf*(K1+1)/(tf + K1*(1-B+B*dl/avgdl))`，`tf=chunk["tf"].get(t,0)`，`dl=chunk["dl"]`。
- `search(index, query, aliases, type_filter="all", topk=20)`（P0 先不接意图/别名，`aliases` 形参留着）：
  1. `topic_tokens = 去重(tokenize(query, **index["config"]))`。
  2. 候选 = 所有 `topic_tokens` 的倒排表并集（**不要遍历全部 N 块**）。
  3. 对候选块算 `final = bm25 + W_HEAD*(命中heading_tokens的topic数/len(topic)) + W_POS*1/(1+position)`（P0 不加 W_TYPE）。
  4. `type_filter in ("prose","code")` 时按 ctype 过滤。
  5. 按 `final` 降序取 `topk`，返回 `[{file,heading_path,ctype,score,text}]`。

**验收**：能对 materials 返回合理排序；`范数` 能召回；特殊字符查询不报错。

**易错点**：
- **IDF 必须 `ln(1+…)`**，否则常见词负 IDF。
- 查询 token **去重**后再遍历（`dict.fromkeys` 保序去重）。
- `len(topic)` 为 0 时标题加分要防除零。

**给 Claude Code 的 prompt**：
```
实现 ranking.py（P0 部分）：常量 W_HEAD=2.0,W_TYPE=1.5,W_POS=0.5,K1=1.5,B=0.75。
- bm25(index, chunk, qtokens)：IDF=ln(1+(N-df+0.5)/(df+0.5))（df 从 index["df"]，缺省视为0），
  对每个 t in qtokens：tf=chunk["tf"].get(t,0)，dl=chunk["dl"]，
  score += IDF*tf*(K1+1)/(tf+K1*(1-B+B*dl/index["avgdl"]))，tf=0 跳过。
- search(index, query, aliases, type_filter="all", topk=20)（本阶段不接意图/别名）：
  topic_tokens=保序去重(tokenizer.tokenize(query, **index["config"]))；
  候选=所有 topic_tokens 在 index["inverted"] 的并集 chunk_id；
  对每个候选块 final=bm25 + W_HEAD*(命中heading_tokens的topic数/len(topic_tokens)) + W_POS*1/(1+position)，
  注意 len(topic_tokens)==0 防除零；
  type_filter in {prose,code} 时按 ctype 过滤；按 final 降序取 topk，
  返回 [{file,heading_path,ctype,score,text}]。
纯标准库 math，Python 3.7+。
```

## 8. 贯穿全程的易错点总清单（对应方案 v2 §15，提交前逐条核对）
