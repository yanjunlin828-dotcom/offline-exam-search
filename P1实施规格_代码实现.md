## 1. ranking.py 最终 API（Task 1.1 + 1.2 完成后）

### 1.1 `search()` 新签名

```python
def search(index, query, aliases, type_filter="all", topk=20):
    """
    Returns: (list[dict], intent)
      - list[dict]: 每条含 {file, heading_path, ctype, score, text}，按 score 降序
      - intent: "prose" | "code" | None
    """
```

**返回值由 `list` 改为 `(list, intent)` tuple**，这是 P1 最重要的 breaking change。

### 1.2 `search()` 内部执行顺序（最终版）

```
1. intent, cleaned_query = detect_intent(query)          ← Task 1.1
2. raw_tokens = tokenize(cleaned_query, **index['config'])
3. topic_tokens = list(dict.fromkeys(raw_tokens))         ← 保序去重
4. 若 topic_tokens 为空 → return [], intent
5. 基础候选：topic_tokens 各自倒排表 chunk_id 并集       ← P0 已有
6. 别名扩展候选（Task 1.2）：
   a. matched_phrases = expand_aliases(set(topic_tokens), cleaned_query, aliases or [], cfg)
   b. 对每个 phrase_tokens（一个短语的 token 列表）：
      phrase_cids = inverted[t1] ∩ inverted[t2] ∩ … （AND，交集）
      candidate_ids |= phrase_cids
      记录 alias_hits[chunk_id].append(phrase_tokens)
7. 对每个 candidate chunk：
   a. type_filter 过滤（prose/code/all）
   b. score_tokens = topic_tokens + alias_hits[chunk_id] 里完整命中的短语 token（去重）
      → 部分命中的别名 token 不计分
   c. final = bm25(index, chunk, score_tokens)
            + W_HEAD * (heading_hits / n_topics)     ← n_topics = len(topic_tokens)
            + W_POS  * (1 / (1 + position))
            + W_TYPE * (1 if intent and ctype == intent else 0)  ← Task 1.1
8. 按 final 降序取 topk
9. return results[:topk], intent
```

### 1.3 `expand_aliases()` 签名与语义

```python
def expand_aliases(topic_token_set, query, alias_groups, cfg):
    """
    Args:
        topic_token_set: set[str]  — cleaned_query 切词后的 token 集合
        query: str                 — cleaned_query 字符串（用于子串匹配）
        alias_groups: list[list[str]] — load_aliases 返回的原始短语组
        cfg: dict                  — {enable_unigram, cjk_ext}

    Returns:
        list[list[str]]  — 每个元素是一个短语的 token 列表
                           来自所有命中的组的所有短语（不含空列表）

    命中规则（对每个 alias_group，任一短语满足即整组命中）：
        条件 A：tokenize(phrase, **cfg) 的 set ⊆ topic_token_set
        条件 B：phrase.lower() in query.lower()
    命中后：对该组每个 phrase，tokenize 后若非空则加入结果。
    """
```

**关键约束（防误召 `method`）**：
- 不能把别名 OR 成散词
- "alternating direction method of multipliers" 需 5 个词全部出现在同一块（AND 交集）才计分
- 只含 `method` 的块不进入候选（因为 phrase_cids 交集为空）

现有实现正确（含 prose/code 意图词、剔除后返回 cleaned_query）。Task 1.1 只需把它接进 `search()`。

### 4.3 `/search` 响应 JSON 结构

```json
{
  "query": "原始查询字符串",
  "intent": "prose",
  "results": [
    {
      "file": "materials/凸优化综述（ADMM·范数·凸性）_全解.md",
      "heading_path": "优化方法综述 > ADMM > 定义",
      "ctype": "prose",

      "text": "ADMM 是交替方向乘子法..."

  ]

```

- `intent`：`"prose"` | `"code"` | `null`
- `results`：按 score 降序，最多 topk（默认 20）条
- `text`：原始正文（前端负责高亮和 HTML 转义）

### 5.4 高亮规则（决策 H，严格执行）

**prose 块**：
```javascript
function highlightProse(text, queryWords) {
    // Step 1: HTML 转义（必须在前）
    let s = text
        .replace(/&/g, '&amp;')
        .replace(/</g, '&lt;')
        .replace(/>/g, '&gt;')
        .replace(/"/g, '&quot;');

    // Step 2: 包 <mark>（转义完成后）
    for (const word of queryWords) {
        if (!word) continue;
        // 转义 regex 特殊字符
        const escaped = word.replace(/[.*+?^${}()|[\]\\]/g, '\\$&');
        s = s.replace(new RegExp(escaped, 'gi'), m => `<mark>${m}</mark>`);
    }
    return s;
}
// queryWords = query.split(/\s+/).filter(Boolean)
// 渲染：elem.innerHTML = highlightProse(result.text, queryWords)
// !! 不走 marked 渲染 !!
```
