## 普通最小二乘（OLS）

### OLS 目标函数与正规方程：XᵀXβ = Xᵀy

OLS 的目标是最小化残差平方和：

$$\min_\beta \|y - X\beta\|_2^2 = \min_\beta \sum_{i=1}^n (y_i - x_i^T\beta)^2$$

对 $\beta$ 求导并令其为零，得到**正规方程（Normal Equations）**：

$$X^TX\hat\beta = X^Ty$$

这是一个 $p \times p$ 的线性方程组（不是 $n \times n$！），维度只和特征数有关，与样本量无关。

### OLS 闭合解：β = (XᵀX)⁻¹Xᵀy

正规方程的显式解：

$$\hat\beta = (X^TX)^{-1}X^Ty$$

**注意：** 这个公式告诉你数学上的答案，但**代码中不能直接翻译为 `np.linalg.inv(X.T @ X) @ (X.T @ y)`**——正确写法是 `np.linalg.solve(X.T @ X, X.T @ y)`（见矩阵运算理论）。

**截距的处理：** 在 $X$ 的最左列加一列全 1 向量，截距 $\beta_0$ 自然融入 $\beta$ 的第一个分量，公式形式不变。

### OLS 的计算瓶颈（大数据场景）

设 $n = 10^9$（10亿样本），$p = 10^4$（1万特征）：

- $X$ 本身大小：$10^9 \times 10^4 \times 8\text{ bytes} \approx 80\text{ TB}$——根本无法载入内存
- 计算 $X^TX$：复杂度 $O(np^2) = O(10^{17})$，单机需 **116 天**

但注意：$X^TX$ 本身只是 $p \times p = 10^4 \times 10^4$，大小约 800 MB，完全可以放进内存。**问题不是结果矩阵大，而是无法一次性读取和处理完整的 $X$。**

## 分布式 OLS

### XᵀX 和 Xᵀy 的可加性

将 $X$ 按行分成 $k$ 块（每块分配给一个节点）：

$$X = \begin{bmatrix}X_1 \\ X_2 \\ \vdots \\ X_k\end{bmatrix} \implies X^TX = \sum_{i=1}^k X_i^TX_i, \quad X^Ty = \sum_{i=1}^k X_i^Ty_i$$

**关键性质：每个 $X_i^TX_i$ 只需要第 $i$ 个节点自己的数据，完全独立计算，结果是 $p \times p$ 的小矩阵，汇总求和即可。**

这是整门课"分布式设计第一范式"：找到统计量的**可加结构** → Map（各节点并行算局部统计量）→ Reduce（主节点求和）→ 仅用小矩阵在主节点完成最终计算。

### 分布式 OLS 算法流程（分块 → 并行 → 聚合 → solve）

```
Map 阶段（k 个节点完全并行）：
  节点 i：拥有 (Xᵢ, yᵢ)
          计算 Mᵢ = XᵢᵀXᵢ（p×p），dᵢ = Xᵢᵀyᵢ（p维）
          返回 (Mᵢ, dᵢ)

Reduce 阶段（主节点）：
  XTX = M₁ + M₂ + ... + Mₖ    ← 每个只有 p×p，通信量与 n 无关
  XTy = d₁ + d₂ + ... + dₖ

求解（主节点，单机）：
  β̂ = np.linalg.solve(XTX, XTy)   ← p×p 的方程组，不求逆
```

**性能关键：** 各节点只向主节点发送 $p \times p$ 的小矩阵，原始大数据永远不离开节点，通信量与 $n$ 无关。

## 岭回归（Ridge Regression）

### 岭回归的动机：p > n 时 OLS 无解

当特征数 $p$ 大于样本数 $n$ 时（如基因组学 $p = 10^4$ 个基因但只有几百个病人），OLS 从数学和工程两个层面都崩溃：

**数学崩溃：** $X$ 是"宽矩阵"（行少列多），秩最多为 $n < p$，所以 $X^TX$（$p \times p$ 矩阵）是**奇异矩阵**，行列式为零，不可逆，OLS 显式解根本不存在。

**工程崩溃：** 当 $p = 10^6$ 时，$X^TX$ 是 $10^6 \times 10^6$ 的矩阵，需要 **8 TB 内存**，即使数学上有解，工程上也无法存储和计算。

### 岭回归目标函数与正规方程：(XᵀX + λI)β = Xᵀy

岭回归在 OLS 损失上加一个 L2 正则化惩罚项：

$$\min_\beta \|y - X\beta\|_2^2 + \lambda\|\beta\|_2^2, \quad \lambda > 0$$

对 $\beta$ 求导令其为零，得到岭回归正规方程：

$$(X^TX + \lambda I)\hat\beta^{ridge} = X^Ty$$

显式解形式：$\hat\beta^{ridge} = (X^TX + \lambda I)^{-1}X^Ty$（代码中同样用 `solve` 而非 `inv`）。

### 正则化参数 λ 的作用（收缩系数、改善条件数）

**为什么 $\lambda I$ 修复了奇异问题？**

若 $X^TX$ 的最小特征值为 $\lambda_{\min}$（当 $p > n$ 时 $\lambda_{\min} = 0$），则 $X^TX + \lambda I$ 的最小特征值为 $\lambda_{\min} + \lambda > 0$。所有特征值均正 → **对称正定** → **必定可逆**。

**λ 的两重作用：**
1. **收缩系数：** 惩罚项压制 $\beta$ 中各分量的绝对值，避免过拟合
2. **改善条件数：** 条件数从 $\kappa(X^TX)$ 降低到 $\kappa(X^TX + \lambda I)$，数值更稳定

**统计代价：** 岭估计是**有偏估计**（而 OLS 是无偏的）。但在 $p > n$ 的高维场景下，OLS 根本不存在，"有偏但存在"总比"无偏但不存在"强。偏差-方差权衡（Bias-Variance Tradeoff）：引入少量偏差，换来方差大幅降低，预测效果更好。

### 岭回归的分布式计算（XᵀX 可加性不变，聚合后加 λI）

分布式 OLS 的流程完全适用于岭回归，唯一区别是在 Reduce 阶段聚合后加上 $\lambda I$：

```
# 与分布式 OLS 完全相同的 Map 阶段
XTX = sum of all local XᵢᵀXᵢ
XTy = sum of all local Xᵢᵀyᵢ

# 唯一的区别：加上正则化
A = XTX + lam * np.eye(p)       # 只在主节点做一次加法
beta_ridge = np.linalg.solve(A, XTy)
```

## 共轭梯度法（CG）

### 直接法 vs 迭代法

求解线性方程组 $Ax=b$ 的方法分两大类：

| | 直接法（LU / Cholesky / QR） | 迭代法（CG / 梯度下降） |
|--|--------------------------|----------------------|
| 形式 | 有限步内给出精确解 | 从初始猜想逐步逼近真解 |
| 优点 | 精度高，算法稳定 | 节省内存，适合超大规模稀疏问题 |
| 缺点 | 必须显式存储完整矩阵 $A$（$O(p^2)$ 内存） | 多次迭代，精度依赖条件数 |

**CG 是两者兼得的特殊方法：** 算法形式类似梯度下降（迭代法），但数学上对 $p \times p$ 正定矩阵保证**最多 $p$ 步内精确收敛**（直接法的性质）。实践中条件数小时往往几十步就足够精确。

### CG 的适用场景（p 极大时唯一可行方案）

岭回归正规方程 $(X^TX + \lambda I)\hat\beta = X^Ty$ 是一个 $p \times p$ 的方程组。即使用 Cholesky，**也需要将完整的 $p \times p$ 矩阵 $X^TX + \lambda I$ 存入内存**。

当 $p = 10^6$（推荐系统中的用户 ID 数量），该矩阵需要 8 TB 内存——Cholesky 直接宣告死亡。

需要一种**连矩阵都不需要显式存储**的方法，这就是**迭代法**的动机。共轭梯度法（CG）是解对称正定方程组的最优迭代法，其特殊之处在于它兼具直接法的精度（理论上有限步收敛）和迭代法的低内存特性。

### Matrix-Free 技巧：(XᵀX + λI)v = Xᵀ(Xv) + λv

CG 每步迭代只需要计算"矩阵与向量的乘积" $Av$（其中 $A = X^TX + \lambda I$），不需要矩阵 $A$ 本身。

$$\underbrace{(X^TX + \lambda I)v}_{A \text{ 乘以向量，}O(p^2)} = \underbrace{X^T(\underbrace{Xv}_{O(np)})}_{\text{先算 }Xv} + \lambda v$$

**关键：** 右边只需两次矩阵-向量乘法（$Xv$ 是 $O(np)$，$X^T(\cdot)$ 是 $O(np)$），总计 $O(np)$。**从不显式构造 $X^TX$（那需要 $O(p^2)$ 存储和 $O(np^2)$ 计算），整个算法的内存需求从 $O(p^2)$ 降到 $O(np)$（存 $X$ 本身），完全可行。**

这种技巧叫做 **Matrix-Free（无矩阵）**：只需要能"执行矩阵与向量的乘法"这个操作，不需要矩阵本身的显式表示。

### CG 算法流程

求解 $Ax = b$（$A$ 对称正定）的完整步骤：

**等价形式：** $Ax=b$ 等价于最小化二次函数 $f(x) = \frac{1}{2}x^TAx - b^Tx$。普通梯度下降在此问题上易"在山谷中走之字形"；CG 每步选取与之前所有方向关于 $A$ **共轭正交**的搜索方向，从而避免重复。

**初始化：** $x_0 = 0$，残差 $r_0 = b$，搜索方向 $p_0 = r_0$

**第 $k$ 步迭代（直到 $\|r_k\|$ 足够小）：**

$$\alpha_k = \frac{r_k^T r_k}{p_k^T A p_k} \qquad \text{（步长）}$$

$$x_{k+1} = x_k + \alpha_k p_k, \qquad r_{k+1} = r_k - \alpha_k A p_k \qquad \text{（更新解与残差）}$$

$$\beta_k = \frac{r_{k+1}^T r_{k+1}}{r_k^T r_k}, \qquad p_{k+1} = r_{k+1} + \beta_k p_k \qquad \text{（更新搜索方向）}$$

> 算法中唯一涉及 $A$ 的操作是乘法 $Ap_k$——无需显式存储 $A$，这正是 Matrix-Free 技巧的切入点。

### CG 的收敛性与条件数

CG 的理论收敛速度取决于矩阵 $A = X^TX + \lambda I$ 的**条件数** $\kappa = \lambda_{\max}/\lambda_{\min}$：

- 条件数越小，收敛越快
- 加入正则化 $\lambda I$ 会**降低条件数**（分子不变，分母增大），从而加速 CG 收敛
- 这是正则化参数 $\lambda$ 的第三个作用：不仅修复奇异、压制系数，还能加速 CG 收敛

理论上 CG 在 $p$ 步内收敛（精确解），实践中条件数小时几十步内就已高精度收敛。

### CG vs Cholesky 选择依据

| | Cholesky | CG |
|--|----------|----|
| 需要显式存 $A$？ | 是，需 $O(p^2)$ 内存 | 否，只需 $O(np)$ |
| 计算代价 | $O(p^3)$ 一次性（$p$ 中等时快） | 每步 $O(np)$，迭代 $k$ 步共 $O(knp)$ |
| 结果精度 | 精确解（在数值误差范围内） | 迭代近似，精度随迭代增加 |
| 适用场景 | $p$ 中等（如 $p < 10^4$），$X^TX$ 可存 | $p$ 极大（如 $p > 10^5$），$X^TX$ 存不下 |

## OLS / Ridge / CG 对比

### 适用场景对比表

| 方法 | 适用场景 | 关键限制 |
|------|---------|---------|
| OLS + Cholesky | $n \gg p$，$X$ 可分块，$X^TX$（$p\times p$）可存 | $p > n$ 时 $X^TX$ 奇异，无解 |
| Ridge + Cholesky | $p > n$，$p$ 中等（$p^2$ 可存） | $p$ 极大时 $p\times p$ 矩阵存不下 |
| Ridge + CG | $p > n$，$p$ 极大 | 需要迭代，收敛速度依赖条件数 |

### 计算复杂度对比

| 方法 | 内存需求 | 时间复杂度 |
|------|---------|-----------|
| OLS（正规方程 + Cholesky） | $O(p^2)$（存 $X^TX$）| $O(np^2)$ 构造 + $O(p^3)$ 分解 |
| Ridge（正规方程 + Cholesky） | $O(p^2)$（存 $X^TX + \lambda I$）| 同上 |
| Ridge + CG（Matrix-Free） | $O(np)$（只存 $X$）| 每步 $O(np)$，迭代 $k$ 步 |
