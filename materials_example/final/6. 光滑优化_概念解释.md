## 梯度下降法（GD）

### GD 算法步骤与学习率选择

**核心思想：** 每步沿着当前梯度的反方向移动。

$$\beta^{(t+1)} = \beta^{(t)} - \alpha \nabla\ell(\beta^{(t)})$$

对 Logistic 回归，梯度 $\nabla\ell = X^T(\hat{p}^{(t)} - y)$，故每步更新：

$$\beta^{(t+1)} = \beta^{(t)} - \alpha \cdot X^T(\hat{p}^{(t)} - y)$$

**单次迭代流程：**
1. 用当前 $\beta^{(t)}$ 计算预测概率 $\hat{p}^{(t)} = \sigma(X\beta^{(t)})$
2. 计算梯度 $g = X^T(\hat{p}^{(t)} - y)$（维度 $p$）
3. 更新 $\beta^{(t+1)} = \beta^{(t)} - \alpha g$

**学习率 $\alpha$ 的影响：**
- $\alpha$ 太大：每步跨越最优点，在最优点两侧来回震荡，无法收敛
- $\alpha$ 太小：每步只前进一点点，需要极多次迭代
- 需要手动调参；线搜索（line search）是自动确定 $\alpha$ 的标准方法

**单步计算复杂度：** $O(np)$——需要计算 $X\beta$（$O(np)$）和 $X^T(\cdot)$（$O(np)$）。

### GD 的收敛条件（凸函数保证）

对于凸函数，GD 从任意初始点出发都会收敛到全局最优。收敛速率为 $O(1/t)$（"线性收敛"），即需要 $O(1/\varepsilon)$ 步才能使误差小于 $\varepsilon$。

### GD 的适用范围与局限（小数据、光滑凸目标）

**适合：** 小规模数据（$n$ 不大），对精度要求高，调试阶段。

**局限：** 大数据时每步扫描全量数据代价太高；对学习率 $\alpha$ 非常敏感；遇到平坦区域（梯度接近零但尚未收敛）速度极慢。

## 牛顿法（Newton's Method）

### 牛顿法原理：用二阶信息近似曲率

**核心思想：** 梯度下降只用一阶信息（坡度方向），牛顿法同时利用二阶信息（曲率），通过二次近似在当前点估计极值位置，一步直接"跳"过去。

**类比：** 梯度下降像蒙着眼睛只靠脚感走下坡；牛顿法像能看到前方地形的弯曲程度（曲率），直接估算最低点位置并一步到位。

### 牛顿方向：Δβ = -H⁻¹∇ℓ

**更新公式：**

$$\beta^{(t+1)} = \beta^{(t)} - \alpha \underbrace{[H(\beta^{(t)})]^{-1} \nabla\ell(\beta^{(t)})}_{\text{牛顿方向 }\Delta\beta}$$

对 Logistic 回归：$H = X^TWX$，$\nabla\ell = X^T(\hat{p} - y)$，故牛顿方向满足线性方程组：

$$X^TWX \cdot \Delta\beta = -X^T(\hat{p} - y)$$

**与 IRLS 的联系：** 牛顿法求解 Logistic 回归等价于迭代地求解加权最小二乘（IRLS，Iteratively Reweighted Least Squares），因为 $H = X^TWX$ 正是加权 OLS 的 Gram 矩阵。

后期可取步长 $\alpha = 1$（纯牛顿步），自适应地决定步长大小，不需要手动调参。

### 牛顿法 vs 梯度下降（收敛速度 vs 单步代价对比）

| 特性 | 梯度下降（GD） | 牛顿法 |
|------|-------------|--------|
| 利用的信息 | 一阶（梯度） | 二阶（Hessian） |
| 收敛速度 | 线性收敛，$O(1/t)$ | **二阶收敛**（误差平方式减小，极快） |
| 单步计算代价 | $O(np)$ | $O(np^2 + p^3)$（构造 $H$ + 求解方程组） |
| 学习率 | 需要手动调节 | 后期可取 $\alpha=1$（自适应） |

### 牛顿法的限制（Hessian 存储 O(p²)，p 大时不可行）

牛顿法需要构造 $p \times p$ 的 Hessian 矩阵 $X^TWX$ 并求逆（或解方程组）：
- 内存：$O(p^2)$。当 $p = 10^4$，Hessian 需要 800MB；$p = 10^5$ 时需要 80GB——彻底超出单机内存
- 计算：$O(p^3)$ 的 Cholesky 分解。当 $p$ 大时灾难性慢

**结论：** 牛顿法只在 $p$ 较小（如 $p < 10^4$）时实用；$p$ 更大时需要 L-BFGS。

## L-BFGS 算法

### L-BFGS 的动机：避免存储完整 Hessian

牛顿法极快但代价太高，GD 代价低但收敛慢。能不能有一种方法，不算 Hessian，却有接近牛顿法的收敛速度？

**拟牛顿法（Quasi-Newton）的核心思路：** 不直接计算 Hessian，而是从迭代过程中梯度的变化来**逐步近似** Hessian 的逆。更新方向形如：

$$\beta^{(t+1)} = \beta^{(t)} - \alpha (B^{(t)})^{-1}\nabla\ell(\beta^{(t)})$$

其中 $B^{(t)}$ 是 Hessian 的近似，通过**拟牛顿方程**约束：

$$B^{(t+1)} s^{(t)} = y^{(t)}$$

- $s^{(t)} = \beta^{(t+1)} - \beta^{(t)}$（参数位移）
- $y^{(t)} = \nabla\ell(\beta^{(t+1)}) - \nabla\ell(\beta^{(t)})$（梯度变化）

**直觉：** 相邻两步的梯度变化 $y^{(t)}$ 携带了曲率信息，用这些信息逐步建立 Hessian 的近似——不需要显式算 $X^TWX$。

### 有限内存近似原理（只保留最近 m 步的曲率信息）

标准 BFGS（Broyden-Fletcher-Goldfarb-Shanno）仍需存储 $p \times p$ 的近似矩阵 $B^{(t)}$，$p$ 大时依然爆内存。

**L-BFGS（Limited-Memory BFGS）改进：** 不存储 $B^{(t)}$ 矩阵本身，只保留**最近 $m$ 步**的位移/梯度变化对 $\{(s^{(t)}, y^{(t)})\}$（通常 $m = 5 \sim 10$）。

通过"两步循环"递推算法，直接从这 $m$ 对向量计算搜索方向 $(B^{(t)})^{-1}g$，无需显式存矩阵。

**存储对比（$p = 10^6$，$m = 10$）：**

| 算法 | 存储复杂度 | 实际内存 |
|------|---------|--------|
| 牛顿法 | $O(p^2)$ | ~8 TB |
| BFGS | $O(p^2)$ | ~8 TB |
| **L-BFGS** | **$O(mp)$** | **~160 MB** |

### L-BFGS 在 Logistic 回归中的使用

L-BFGS 对外只需要一个接口：**给定 $\beta$，返回损失值 $\ell(\beta)$ 和梯度 $\nabla\ell(\beta)$。** 优化器内部自己管理曲率近似。

```python
from scipy.optimize import fmin_l_bfgs_b

# func 只需返回 (损失值, 梯度向量)
result = fmin_l_bfgs_b(func=get_loss_and_grad, x0=beta_init)
```

**分布式版本：** Map-Reduce 负责计算梯度，L-BFGS 负责优化策略，两者完全解耦：

```
L-BFGS 优化器（头节点）
  ↓ 请求：给我 β 点的损失和梯度
Ray Map-Reduce（K 个工作节点并行计算局部梯度）
  ↓ 返回：全局 ℓ(β) 和 ∇ℓ(β)
L-BFGS 优化器 → 利用历史曲率更新 β → 重复
```

### GD / Newton / L-BFGS 三方对比（内存 / 速度 / 适用场景）

| 特性 | GD | 牛顿法 | L-BFGS |
|------|-----|--------|--------|
| 利用信息 | 一阶（梯度） | 二阶（Hessian） | 近似二阶 |
| 收敛速度 | 线性（慢） | 二阶（极快） | 超线性（快） |
| 单步计算 | $O(np)$ | $O(np^2 + p^3)$ | $O(np + mp)$ |
| 存储需求 | $O(p)$ | $O(p^2)$ | $O(mp)$（$m \ll p$） |
| 学习率 | 需要手动调 | 后期自适应 | 内置线搜索 |
| 适合规模 | 任意（尤其超大） | 小规模（$p$ 小） | **中大规模首选** |
| 分布式扩展 | 容易 | 困难（Hessian 通信） | 容易（只通信梯度） |

## 随机梯度下降（SGD）

### SGD 的核心思想（单样本近似全量梯度）

**问题根源：** 全量 GD 每步扫描全部 $n$ 个样本，$n = 10^9$ 时一次迭代需要几十分钟。

**SGD 的解决思路：** 每次随机抽取一个极小子集（**Mini-batch** $B$，大小 $|B|$ 通常 32~1024），用这个小批量的平均梯度近似全量梯度，立刻更新：

$$\beta^{(t+1)} = \beta^{(t)} - \alpha \cdot \underbrace{\frac{1}{|B|}\sum_{i \in B}\nabla\ell_i(\beta^{(t)})}_{\nabla\ell_B(\beta^{(t)})}$$

**为什么可行（无偏性）：** $B$ 从 $n$ 个样本随机均匀抽取，故：

$$\mathbb{E}[\nabla\ell_B(\beta)] = \frac{1}{n}\sum_{i=1}^n\nabla\ell_i(\beta) = \nabla\ell(\beta)$$

Mini-batch 梯度是全量梯度的**无偏估计**——方向平均而言正确，只是有随机噪声。

### Mini-batch SGD（批次大小对方差和速度的权衡）

| $|B|$ 大小 | 梯度噪声 | 单步时间 | 一个 epoch 更新次数 |
|----------|---------|--------|-----------------|
| 1（纯 SGD） | 最大 | 最快 | $n$ 次 |
| 32~256（常用） | 适中 | 适中 | $n/|B|$ 次 |
| $n$（全量 GD） | 零（精确） | 最慢 | 1 次 |

**扩展批次定律：** 批次大小增加 $K$ 倍，学习率也应增加 $K$ 倍（线性扩展规则），以维持相同的收敛行为。

### Robbins-Monro 收敛条件（学习率衰减规则：Σαₜ=∞, Σαₜ²<∞）

SGD 使用的是有噪声的近似梯度，要保证收敛到真正的极值点，学习率序列 $\{\alpha_t\}$ 必须满足：

$$\sum_{t=1}^{\infty}\alpha_t = \infty \quad \text{（步长之和无穷大，保证能走到足够远）}$$

$$\sum_{t=1}^{\infty}\alpha_t^2 < \infty \quad \text{（步长平方之和有限，保证最终停下来不震荡）}$$

**两个条件的直觉：** 第一个条件保证"能到达"，第二个条件保证"能停下"——步长必须越来越小，最终衰减到零。

**典型选择：** $\alpha_t = 1/t$（满足条件但衰减过快）；$\alpha_t = 1/\sqrt{t}$（常用）；深度学习实践中常用常数学习率配合 Adam。

### SGD 的噪声特性（逃离鞍点）

SGD 的随机扰动是一把双刃剑：
- **坏处：** 参数更新路径为"锯齿形"而非平滑下降，需要更多次迭代
- **好处：** 梯度噪声帮助逃离鞍点（saddle point）——全量 GD 在鞍点处梯度精确为零会卡住，而 SGD 的随机扰动会推离鞍点

## 动量法与 Adam 优化器

### 动量法（Momentum）：给 SGD 加惯性

**SGD 的问题：** Mini-batch 不同导致梯度方向有随机噪声，更新路径在垂直于最优方向上来回震荡。

**动量法的解决思路：** 引入"速度向量" $v^{(t)}$（历史梯度的指数加权累积），用它代替瞬时梯度来更新参数，类似物理中的惯性：

$$v^{(t)} = \gamma v^{(t-1)} + \alpha \cdot \nabla\ell_B(\beta^{(t)})$$

$$\beta^{(t+1)} = \beta^{(t)} - v^{(t)}$$

$\gamma \in (0,1)$ 是动量系数（典型值 0.9），控制历史方向的保留程度。

**展开后的直觉：** $v^{(t)} = \sum_{k=0}^t \gamma^{t-k} \alpha g^{(k)}$，即所有历史梯度的指数加权平均——越近的梯度权重越高，越久远的梯度影响越小。

**两个效果：**
1. **抑制震荡：** 在垂直于最优路径的方向，历史梯度正负相消，合力减弱
2. **加速平坦区域：** 在方向一致的维度，梯度持续叠加，速度越来越大

### Adam：自适应学习率 + 动量的组合

**Adam（Adaptive Moment Estimation）** 把动量（一阶矩）和自适应学习率（二阶矩）合并：

**第一步：更新一阶矩 $m^{(t)}$（梯度的指数加权平均，代表"方向"）**

$$m^{(t)} = \beta_1 m^{(t-1)} + (1-\beta_1)g^{(t)}$$

**第二步：更新二阶矩 $v^{(t)}$（梯度平方的指数加权平均，代表"波动幅度"）**

$$v^{(t)} = \beta_2 v^{(t-1)} + (1-\beta_2)g^{(t)} \odot g^{(t)}$$

$\odot$ 为逐元素相乘。$v^{(t)}$ 的每个维度越大，说明该维度梯度历来波动越剧烈，学习率应缩小以防震荡。

**第三步：偏差修正 + 参数更新**

由于 $m^{(0)} = v^{(0)} = 0$，训练早期两个矩估计系统偏小（被零初始值拉低），需要修正：

$$\hat{m}^{(t)} = \frac{m^{(t)}}{1-\beta_1^t}, \quad \hat{v}^{(t)} = \frac{v^{(t)}}{1-\beta_2^t}$$

$$\boxed{\beta^{(t+1)} = \beta^{(t)} - \frac{\alpha}{\sqrt{\hat{v}^{(t)}} + \varepsilon} \odot \hat{m}^{(t)}}$$

$\frac{\alpha}{\sqrt{\hat{v}^{(t)}} + \varepsilon}$ 是每个维度**各自的有效学习率**：波动大的维度学习率小，波动小的维度学习率大。

### Adam 的参数（β₁, β₂, ε）与偏差修正

| 超参数 | 含义 | 推荐值 |
|------|------|------|
| $\alpha$ | 全局学习率 | 0.001 |
| $\beta_1$ | 一阶矩衰减系数 | 0.9 |
| $\beta_2$ | 二阶矩衰减系数 | 0.999 |
| $\varepsilon$ | 防止除以零的数值稳定常数 | $10^{-8}$ |

默认值在绝大多数深度学习任务上几乎不需要调整。

**偏差修正的消退：** $t$ 增大后，$\beta_1^t \to 0$，$1-\beta_1^t \to 1$，修正项自动消失——只在训练早期起作用。

### Adam vs SGD 的实践选择

- **深度学习：** 几乎默认使用 Adam（自适应学习率省去了手动调参的麻烦）
- **凸优化（如 Logistic 回归）：** SGD + 学习率衰减也足够；L-BFGS 通常更快（更少迭代次数）
- **不推荐 Adam 的情况：** 某些理论研究表明 Adam 在凸问题上的收敛性不如 SGD 有保证；泛化性有时不如带动量的 SGD

## 分布式 Mini-batch SGD

### 分布式 Mini-batch SGD 算法流程（广播-计算-聚合-更新）

**设置：** 集群有 1 个头部节点 + $K$ 个工作节点，数据均匀切分为 $K$ 份。

**五步迭代（每轮重复）：**

```
① 数据切分（只做一次）：
   全量数据 {xᵢ, yᵢ} 均匀切成 K 份，每节点保留自己的 n/K 条数据

② 广播（每轮）：
   头节点将当前参数 β⁽ᵗ⁾ 发送给所有 K 个工作节点

③ 局部采样（工作节点）：
   各节点从本地数据中随机抽取 Mini-batch Bₖ

④ 并行计算局部梯度（工作节点，全部并行）：
   节点 k：gₖ = ∇ℓ_{Bₖ}(β⁽ᵗ⁾) = (1/|Bₖ|) Σᵢ∈Bₖ ∇ℓᵢ(β⁽ᵗ⁾)

⑤ 聚合 + 更新（头节点）：
   G = (1/K) Σₖ gₖ
   β⁽ᵗ⁺¹⁾ = AdamUpdate(β⁽ᵗ⁾, G)
```

**Adam 状态（$m^{(t)}, v^{(t)}$）维护在头节点**——只有头节点做参数更新，工作节点只返回梯度。

### 通信瓶颈：梯度同步的代价

分布式 SGD 中，每步计算极快（Mini-batch 几毫秒），但参数更新极频繁：工作节点算完一个 batch 就要把梯度发给头节点，等待参数更新后才能继续下一步。

**"计算饥饿"：** 工作节点大量时间在等待网络传输，而非真正计算。

| | 全量方法（L-BFGS） | 随机优化（SGD/Adam） |
|---|---|---|
| 每次迭代计算量 | 重（扫描全量数据） | 轻（Mini-batch，毫秒级） |
| 每次迭代通信量 | 轻（几分钟才交换一次） | 重（每秒可能交换多次参数） |
| 通信是否成为瓶颈 | 否 | **是** |

### 批次大小扩展定律（线性扩大批次配合学习率缩放）

将 $K$ 台机器的批次合并等效于在单机上用 $K \times |B|$ 的大批次。实验表明：批次扩大 $K$ 倍，学习率也应线性扩大 $K$ 倍，可维持相似的收敛曲线（Linear Scaling Rule）。

## 参数服务器（Parameter Server）架构

### PS 架构：Server 节点与 Worker 节点的职责划分

参数服务器将集群分为两类角色：

- **参数服务器（PS）：** 专门存储和更新模型参数 $\beta$，汇总各 Worker 推送的梯度后更新参数并广播
- **工作节点（Worker）：** 专门计算局部梯度，不存储完整参数

**工作流程：**
```
Worker 1 → push g₁ → PS → 汇总梯度 → 更新 β → 广播新 β
Worker 2 → push g₂ ↗
Worker 3 → push g₃ ↗
```

每个 Worker 循环：拉取 $\beta$ → 算本地梯度 → 推送梯度 → 重复。

### 分片参数服务器（Sharded PS）

**单 PS 的瓶颈：** $K$ 个 Worker 同时向 PS 发送梯度，PS 的网络带宽被打满（$K$ 倍压力集中在一台机器上）。

**改进：** 把参数向量 $\beta$（$p$ 维）分成 $S$ 块，分别由 $S$ 台 PS 管理，流量被均摊。Worker 将梯度也拆分，各对应部分发给对应的 PS。

**最适合稀疏场景：** 推荐系统中，每个 Mini-batch 只涉及极少数用户/物品的嵌入向量，梯度绝大部分为零——工作节点只传非零部分，通信量大幅降低。

### PS 的局限（集中式瓶颈、单点故障）

| 问题 | 说明 |
|------|------|
| 带宽单点瓶颈 | 所有 Worker 流量汇聚在 PS，节点多时 PS 成为瓶颈 |
| 单点故障 | PS 宕机则整个训练失败 |
| 角色不对等 | Worker 和 PS 职责分离，部署和维护更复杂 |

## 环形全归约（Ring All-Reduce）

### Ring All-Reduce 原理：去中心化通信

**革命性思想：** 彻底去掉中心节点，所有 $K$ 个节点**角色完全对等**，排成逻辑圆环，通过环上两圈传递让每个节点都得到全局梯度 $G = \sum_{k=1}^K g_k$。

**两阶段操作：**

**第一圈（Scatter-Reduce / 求和圈）：** 每个节点把自己梯度向量的第 $j$ 块发给右邻居，同时从左邻居接收并累加一块，共传 $K-1$ 步。结束后，每个节点恰好持有其中一个块的完整全局求和。

**第二圈（All-Gather / 广播圈）：** 把上圈算好的"完整求和块"继续在环上传一圈（$K-1$ 步）。结束后，每个节点都拥有所有块的完整全局梯度。

### All-Reduce 两阶段（Scatter-Reduce + All-Gather）

**总通信量：** 每个节点发送和接收的数据量均约为 $2p$（与节点数 $K$ **几乎无关**！）。

对比：参数服务器中，PS 需要接收 $K$ 个 $p$ 维向量（共 $Kp$），PS 的通信压力随 $K$ 线性增长。

### Ring All-Reduce vs PS 通信效率对比

| | 参数服务器 | Ring All-Reduce |
|---|---|---|
| 有无中心节点 | 有（PS 是中心） | 无，所有节点对等 |
| 单节点通信量 | PS：$O(Kp)$；Worker：$O(p)$ | 每节点均为 $O(p)$ |
| 扩展性 | 差（$K$ 大时 PS 成瓶颈） | **好**（通信量不随 $K$ 增加） |
| 适合场景 | 稀疏模型（推荐系统 Embedding） | **密集模型**（深度学习） |

### 生产环境工具（NCCL / Horovod / PyTorch DDP）

Ring All-Reduce 底层涉及 GPU 显存直接传输等硬件优化，手动实现极难，生产环境使用成熟工具：

- **NVIDIA NCCL：** NVIDIA 专为 GPU 互联优化的通信库，实现极速环形全归约
- **PyTorch DDP（DistributedDataParallel）：** 底层调用 NCCL，用户只需几行代码装饰即可分布式训练
- **Horovod（Uber）：** 框架无关的 Ring All-Reduce 库，支持 TensorFlow/PyTorch
- **DeepSpeed（微软）：** 在 All-Reduce 基础上进一步对参数和优化器状态分片存储（ZeRO）

**Ray 的定位：** Ray 本身不做 Ring All-Reduce（其对象存储设计不适合高频小通信），但 Ray Train 作为**调度和容错层**，启动 GPU 进程后把通信权移交给 NCCL。

## 优化算法选择依据

### 按问题规模选优化器（小 p 牛顿 / 大 p L-BFGS / 超大数据 SGD）

| 策略 | 适用场景 | 核心优势 | 核心限制 |
|------|---------|---------|---------|
| 全量 GD | 小数据集，调试阶段 | 收敛路径平滑，分析方便 | 大数据时每步代价太高 |
| 牛顿法 | 小规模（$p < 10^4$），高精度需求 | 二阶收敛，迭代次数极少 | $p$ 大时 Hessian 存储爆炸 |
| **L-BFGS** | $p$ 中等，大数据，传统 ML | 比 GD 快，比牛顿法省内存 | 仅适合光滑凸问题 |
| 分布式 SGD/Adam | 深度学习，极大规模，$p$ 超大 | 每步快，扩展性强 | 通信频繁，需要架构支持 |

### 按分布式架构选通信方式（PS vs Ring All-Reduce）

| 场景 | 推荐架构 | 原因 |
|------|---------|------|
| 深度学习（密集梯度） | **Ring All-Reduce** | 无中心瓶颈，带宽均摊，扩展到千节点 |
| 推荐系统（稀疏 Embedding） | **分片参数服务器** | 只传非零梯度，通信量大幅减少 |
| 小集群（< 10 节点） | 单参数服务器 | 简单，单点瓶颈不明显 |

### 深度学习默认组合：Adam + Ring All-Reduce

**Adam** 解决了"学习率难调"的问题（自适应，几乎不需要手动调参），**Ring All-Reduce** 解决了"通信瓶颈"问题（无中心节点，带宽均摊）。

两者组合是当今大模型训练（GPT、BERT、LLaMA 等）的**事实标准**。
